# Contributing Rules

We are excited to open this repository to public contributions and welcome developers of all skill levels to help improve the project. To ensure a smooth collaboration process, please take a moment to review our contribution guidelines and Git naming conventions outlined below.

## Contibution & Git Pipelines

To keep our codebase clean and manageable, please adhere to the following rules when contributing:

- **Branch naming**: Name your branch using a word or two that describes feature or functionality that you are bringing to the project. Example: `docs` branch for anything realted to project documentation, `auth` for anything auth related, etc.
- **Commit naming**: For every commit use a prefix: + brief title(optionally add a small description). Common prefixes: 
    - `feat` for new features/functionality
    - `fix` for logic fixes
    - `chore` for chore tasks(config files, makefiles, etc.)
    - `docs` for documentation related
- **Pull requests (PRs)**: After finishing your work, open a pull request, ensure your branch is reabsed with the latest `main` branch and all tests pass. Provide an explicit description and a title for every PR opened. Every pull request is going to be review by the main contributors.
